# AIO (All-In-One) - 要件定義書

## 1. プロジェクト概要

### 1.1 プロダクト名
**AIO** (All-In-One)

### 1.2 コンセプト
複数のWebベースコミュニケーションサービスを1つのデスクトップアプリに統合し、サービスごとに独立したログインセッションを保持したまま切り替えられるワークスペース管理アプリケーション。

### 1.3 解決する課題
- 複数のコミュニケーションツール(Slack, Chatwork, Facebook Messengerなど)をブラウザのタブで開くと管理が煩雑になる
- 同一サービスで複数アカウント(例: プロジェクトAのSlack / プロジェクトBのSlack)を使い分ける際、ログイン/ログアウトの切り替えが面倒
- ブラウザのCookieが共有されるため、同一サービスの複数アカウント同時利用が困難

### 1.4 類似プロダクト
- [Biscuit](https://eatbiscuit.com/)
- [Franz](https://meetfranz.com/)
- [Ferdi](https://getferdi.com/)
- [Station](https://getstation.com/)

## 3. 技術アーキテクチャ

### 3.1 コア技術
- **Electron BrowserView / webview** を各サービス・各アカウントごとに独立したインスタンスとして生成
- **session.fromPartition()** によりアカウントごとにCookie/セッション/LocalStorageを完全分離
- セッションデータはアプリ終了後も永続化(`persist:` パーティションにより再起動時に再ログイン不要)

### 3.2 アーキテクチャ
- メインプロセス: Electron (Node.js) — ウィンドウ管理、セッション管理、IPC
- レンダラープロセス: React + TypeScript — UI描画
- プロセス間通信: Electron IPC (contextBridge + preload)

## 4. 機能要件

### 4.1 サービス管理

#### 4.1.1 プリセットサービス
以下のサービスをプリセットとして提供する:

| サービス名 | URL | アイコン |
|------------|-----|----------|
| Slack | https://app.slack.com/ | 同梱 |
| Chatwork | https://www.chatwork.com/ | 同梱 |
| Facebook Messenger | https://www.messenger.com/ | 同梱 |
| Discord | https://discord.com/app | 同梱 |
| Microsoft Teams | https://teams.microsoft.com/ | 同梱 |
| Gmail | https://mail.google.com/ | 同梱 |
| Google Drive | https://drive.google.com/ | 同梱 |
| GitHub | https://github.com/ | 同梱 |

#### 4.1.2 カスタムサービス
- ユーザーが任意のURLを指定してカスタムサービスを追加可能
- サービス名、URL、アイコン(任意)を設定可能

### 4.2 アカウント管理

#### 4.2.1 マルチアカウント
- 1つのサービスに対して複数のアカウント(インスタンス)を追加可能
- 各アカウントは**独立したセッション**を保持(`session.fromPartition()` による完全分離)
- アカウントごとにラベル(例:「プロジェクトA」「個人用」)を設定可能
- アカウントごとにカラーラベルを設定可能(視覚的な区別)

#### 4.2.2 セッション永続化
- `persist:<account-id>` パーティションにより、アプリ終了後もセッションを自動保存
- アプリ再起動時にセッションを復元し、再ログイン不要とする
- 個別アカウント単位でセッションクリア(ログアウト)可能

### 4.3 UI/UX

#### 4.3.1 デザイン方針
- **シンプル・クリーン・洗練**: Tailwind CSSライクなミニマルデザイン
- ホワイトモードのみ(初期リリース)
- 配色: 白背景 + グレー系ボーダー + 控えめなアクセントカラー
- フォント: システムフォント(-apple-system, Segoe UI)
- 余白を十分に取り、情報密度を抑えた読みやすいレイアウト
- ホバー/アクティブ状態は淡いグレーのハイライトで表現

#### 4.3.2 サイドバー
- 左側にサービス/アカウントの一覧をアイコンバー形式で表示
- サービスアイコン + アカウントラベル(ツールチップ)で表示
- ドラッグ&ドロップで並び替え可能
- サイドバーの折りたたみ(トグル)対応
- 未読バッジの表示(対応サービスのみ)
- サイドバー内の項目は上から順に `Cmd/Ctrl + 1`, `Cmd/Ctrl + 2`, ... `Cmd/Ctrl + 9` でアクセス可能

#### 4.3.3 メインコンテンツエリア
- サイドバーで選択したアカウントのWebViewを表示
- 非アクティブなWebViewはバックグラウンドで維持(切り替え時にリロード不要)
- ページ内の戻る/進む/リロード操作

#### 4.3.4 ワークスペース(グルーピング)
- アカウントをグループ化して「ワークスペース」として管理可能
- 例:「仕事A」ワークスペース = Slack(A社) + Chatwork(A社) + Gmail(A社)
- ワークスペース単位での一括切り替え

### 4.4 通知

- 各WebView内のWeb通知(Notification API)をOS標準通知に連携(macOS / Windows両対応)
- 通知クリック時に該当アカウントのWebViewをアクティブ化
- アカウント単位で通知のON/OFF設定
- Do Not Disturb(おやすみモード / 集中モード)対応

### 4.5 キーボードショートカット

> `CmdOrCtrl` = macOSでは `Cmd`、Windowsでは `Ctrl`

| 操作 | ショートカット |
|------|----------------|
| サイドバーN番目のアカウントに切り替え | `CmdOrCtrl + 1〜9` |
| 次のアカウントに切り替え | `CmdOrCtrl + ]` |
| 前のアカウントに切り替え | `CmdOrCtrl + [` |
| サイドバーのトグル | `CmdOrCtrl + Shift + S` |
| ページリロード | `CmdOrCtrl + R` |
| 戻る | `CmdOrCtrl + ←` |
| 進む | `CmdOrCtrl + →` |
| 設定を開く | `CmdOrCtrl + ,` |
| ページ内検索 | `CmdOrCtrl + F` |

### 4.6 設定

- 一般設定
  - ログイン時に自動起動
  - メニューバー/システムトレイ常駐
  - 起動時に前回のワークスペースを復元
- プライバシー
  - アカウントごとのセッションデータ削除
  - 全データの一括削除

## 5. 非機能要件

### 5.1 パフォーマンス
- アカウント切り替え時のレスポンス: 100ms以内(体感即座)
- メモリ使用量: アカウント1つあたり最大200MB目安
- 非アクティブなWebViewのメモリ圧縮/サスペンド機能(10分以上非アクティブ時)

### 5.2 セキュリティ
- セッションデータはアプリのユーザーデータディレクトリ内に保存
- アカウント間のデータ分離(`session.fromPartition()` による完全分離)
- contextIsolation: true, nodeIntegration: false(Electronセキュリティベストプラクティス準拠)

### 5.3 安定性
- WebViewクラッシュ時の自動復旧(他のアカウントへの影響なし)
- アプリ異常終了時のセッション復元

### 5.4 アクセシビリティ
- キーボードナビゲーション対応
- スクリーンリーダー対応
